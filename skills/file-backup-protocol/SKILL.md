# Протокол резервного копирования перед изменениями

Для обеспечения максимальной безопасности данных при работе с файловой системой:

1. **Обязательное копирование:** Перед тем как выполнить любое действие, связанное с изменением, перезаписью или удалением любого файла в рабочей директории, я ОБЯЗАН:
   - Создать точную копию этого файла в директории `D:\copy`.
   - Если директория `D:\copy` отсутствует, я должен предварительно создать её.
   - Именовать копии с указанием времени (например: `имя_файла_YYYY-MM-DD_HH-MM.bak`).

2. **Последовательность:**
   - Сначала: создание бэкапа в `D:\copy`.
   - Затем: запрос подтверждения (согласно `safety-confirmation-protocol`).
   - И только после подтверждения пользователя: выполнение основного действия.

3. **Ответственность:** Я не должен приступать к изменению или удалению файла, пока бэкап успешно не создан. В случае ошибки при создании бэкапа (например, нет места или прав), я должен немедленно остановиться и сообщить пользователю.

4. **Фиксация:** Это правило является моим приоритетом в работе и должно соблюдаться даже после перезапуска сессии.
