# Отчёт о проделанной работе (Go-Live сессия)

## 1. Исправление критических ошибок
- **Баг БД:** Сделали поле `reporter_telegram_id` в таблице `reports` необязательным (nullable), чтобы сайт мог отправлять анонимные жалобы без ошибки SQLite.
- **Уведомления админа:** Добавлен `ADMIN_CHAT_ID` в `.env`. Реализована функция `send_admin_notification` в `backend/main.py`, которая отправляет уведомление в Telegram при скрытии объявления (после 3 жалоб).
- **Безопасность:** Внедрена проверка бан-листа в боте (`/add` теперь проверяет `banned_users`).
- **Исправление багов:** Устранена ошибка отступов (IndentationError) и пропущенные импорты в `bot/main.py`.

## 2. Новый функционал
- **Команда /my:** Реализована возможность управления своими объявлениями (удаление и продление) через инлайн-кнопки.
- **Сжатие фото:** Подключена библиотека `Pillow`, добавлено автоматическое сжатие изображений при загрузке (макс. ширина 1200px, качество 80%).

## 3. Инфраструктура и деплой
- **Systemd:** Созданы и настроены юниты `klapa-bot.service` и `klapa-backend.service` для автоматического управления процессами.
- **CI/CD:** Обновлён GitHub Actions (`deploy.yml`) для корректного перезапуска сервисов через `systemctl`.
- **Nginx:** Установлен и настроен Nginx как Reverse Proxy (проксирует трафик с 80-го порта на FastAPI 8000).
- **Firewall:** Настроен `ufw` для защиты сервера (разрешены порты 22, 80).

## 4. Финализация деплоя
- **Nginx:** Настроен в качестве Reverse Proxy (прослушивает порт 80, проксирует на порт 8000).
- **CI/CD:** В `deploy.yml` внесены изменения: теперь деплой использует `systemctl` для перезапуска сервисов, исключив использование `start.sh`.
- **Идемпотентность:** В `backend/schema.sql` добавлен `IF NOT EXISTS` для предотвращения ошибок при перезапуске базы данных.
- **Статус:** 6 июля 2026 г. сайт успешно запущен, настроен автозапуск через `systemd`, работает через Nginx.
